# 媒体分段合成编辑器（Media Segment Composer）

**状态**：草稿 / 待评审  
**日期**：2026-04-18  
**优先级**：P2（PPT2Video 稳定后跟进）

---

## 定位

**专用于 PPT 形态的视频结构**：每一页对应一张图片 + 一段旁白音频 + 一段文字，按页序线性合成视频。适合讲解类、科普类、演示类内容。

> 对话类节目（多人对话、访谈等非线性结构）将由独立的功能界面处理，不在本工具范围内。

---

## 核心工作流

### 第一步：批量导入素材

三类素材**独立批量选择**，按文件名顺序自动 1:1 匹配：

| 素材类型 | 选择方式 | 匹配规则 |
|---------|---------|---------|
| 文字（.txt） | 批量选择文件 / 选择文件夹 | 按文件名排序，第 N 个文字对应第 N 段 |
| 音频（.mp3/.wav） | 批量选择文件 / 选择文件夹 | 按文件名排序，第 N 个音频对应第 N 段 |
| 图片（.png/.jpg） | 批量选择文件 / 选择文件夹 | 按文件名排序，第 N 张图对应第 N 段 |

三类素材数量可以不相等，以最长的为准，缺少的一方留空。

### 第二步：手工编辑每个卡片

批量导入后生成分段卡片列表（纵列滚动），每张卡片可以：
- 替换/删除图片、音频、文字
- 调整本段在列表中的顺序（上移/下移）
- 删除本段

### 第三步：生成缺失音频

对文字存在但音频为空的分段：
- **批量生成**：一键为所有缺失音频的分段调用 TTS
- **单个生成**：在卡片内单独触发

TTS 使用现有 `core.tts_fish` 管道，声音 ID 在顶部全局配置。

### 第四步：合成输出

所有分段就绪后，点「合成视频」输出 MP4。

## 数据模型

```python
@dataclass
class MediaSegment:
    id: str                     # uuid
    text: str                   # 旁白/字幕文字
    image_path: str | None      # 图片路径
    audio_path: str | None      # MP3 路径；None = 待生成或无
    # 扩展预留（Phase 2）
    extra_overlays: list        # 视频叠层、音效等，MVP 留空

@dataclass
class ComposerProject:
    segments: list[MediaSegment]
    voice_id: str
    enable_subtitles: bool
    output_path: str
    resolution: tuple[int, int]  # 默认 (1920, 1080)
```

项目以 JSON 序列化保存到工作目录，支持断点续做。

## 扩展预留（Phase 2）

数据模型中 `extra_overlays` 字段为空列表，MVP 不实现，UI 中每张卡片底部预留「+ 添加叠层」入口（灰色不可点），Phase 2 启用：

- 视频片段叠层（画中画 / 全覆盖）
- 音效混入
- 多图轮播（一段音频对应多张图依次切换）
- 转场效果

---

## 与 PPT2Video 的关系

PPT2Video 以幻灯片文件为驱动，自动导出分段；本工具以素材文件夹为驱动，手动组织分段。两者底层合成逻辑共用，互补而非竞争。

未来 PPT2Video 可以「导出到合成编辑器」做进一步精调。

---

## 开发顺序（MVP）

1. 数据模型 + JSON 序列化
2. 批量导入逻辑（1:1 顺序匹配）
3. 分段卡片列表 UI（折叠/展开、排序、删除）
4. TTS 生成集成（批量 + 单个）
5. 合成输出
