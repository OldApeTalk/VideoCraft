# 08 · 产品战略与功能分层设计

## 背景

VideoCraft 当前阶段目标是**市场试探**：快速触达用户、验证需求、积累反馈，
再决定下一阶段的投入方向。功能设计和优先级排序以此为出发点。

---

## 用户画像：自媒体创作者

**核心目标用户群**：自媒体 Up 主、视频博主、内容创作者

| 维度 | 描述 |
|------|------|
| 主要需求 | 处理外语视频（下载→转录→翻译→烧录字幕） |
| 高频操作 | 批量视频分割、格式转换、字幕处理 |
| 内容生产辅助 | 标题生成、分段标注、摘要提炼 |
| 付费意愿 | 低（不愿为单一功能付订阅月费，但愿意自付 API 费控制成本） |
| 技术能力 | 中等（能配置 API Key，但不想用命令行） |

**设计原则**：所有功能的 UI 文案、文档表述、交互设计，均以自媒体创作流程为中心出发点。

## 功能分层

### 第一层：免费功能（无需任何 API Key）

依赖本地 FFmpeg + yt-dlp，零外部费用，是吸引用户的基础门槛。
**市面上同类工具多以订阅制或按次付费方式提供，免费层本身就是竞争优势。**

| 功能 | 状态 |
|------|------|
| 视频下载（yt-dlp 图形封装） | ✅ 已有 |
| 字幕烧录到视频 | ✅ 已有 |
| 按时间戳分割视频 | ✅ 已有 |
| 自动均分视频 | ✅ 已有 |
| 视频片段提取 | ✅ 已有 |
| 提取 MP3 音频 | ✅ 已有 |
| 音量调整 | ✅ 已有 |
| MP3 码率转换 | ✅ 已有 |
| SRT 文字提取 | ✅ 已有 |
| 按时间戳提取段落 | ✅ 已有 |
| 视频水印添加 | 🔲 待开发 |
| 视频拼接合并 | 🔲 待开发 |

**方向**：免费层功能越丰富，用户粘性越高。新功能优先考虑能否做成纯本地处理。

### 第二层：AI 增强功能（用户自带 API Key）

目前挖掘较浅，现有功能覆盖了核心流程，但围绕自媒体创作场景仍有大量空间。

**现有 AI 功能：**

| 功能 | 所需 Key | 状态 |
|------|---------|------|
| 语音转字幕 | Lemonfox | ✅ 已有 |
| 字幕翻译 | Gemini / Groq / DeepSeek | ✅ 已有 |
| 生成视频分段标题 | Gemini | ✅ 已有 |
| 精炼分段描述 | Gemini | ✅ 已有 |

| 文字转视频 | Gemini + Google Cloud TTS | ✅ 已有 |

**待挖掘方向（围绕自媒体痛点）：**

| 方向 | 描述 | 优先级 |
|------|------|--------|
| AI 字幕校对/优化 | 转录结果常有错误，AI 自动修正 | 高 |

| Highlight 片段提取 | AI 识别视频精华片段时间点 | 低 |

## 暂不考量的方向

### 云化 / SaaS
视频处理极度消耗服务器算力，云化面临并发、等待、成本三重问题，当前阶段不值得投入。

### Docker 分发
Docker 对桌面 GUI 应用不友好，目标用户不具备使用 Docker 的技术背景。

### 跨平台（短期）
当前专注 Windows Portable 做稳。Mac 用户有需求时，优先考虑本地 Web Server 方案（Gradio/Streamlit），而非直接云化。

---

## 开发优先级原则

1. **稳定优先**：现有功能跑通、跑稳，比新增功能更重要
2. **免费层先行**：新功能优先做成无 Key 可用的版本
3. **围绕自媒体流程**：每个新功能都应能回答"自媒体用户在哪个场景用到这个"
4. **AI 功能透明**：清晰标注每个 AI 功能需要哪个 Key，不给用户制造困惑
